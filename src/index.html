How to use PayBatch

PayBatch builds the bulk-payment import file your bank expects, so you can skip the manual spreadsheet. It currently supports Santander Connect (with more banks planned).

1. Choose your output format

At the top of Build batch, pick the format your bank (Santander) expects:

  • Bacs import (.txt) — Santander Connect's official Bacs payment import file. Needs your debit account, a payment date, and a file sequence number. Choose Single (BACS) for one or more independent payments, or Multiple (MULTIBACS) when every payment shares one debit account and date (reference then becomes mandatory).
  • Mixed payments (.txt) — the wide 85-column layout. Only a payment date and the beneficiary rows are needed.

2. Add payments

  • Use the Add a payment form on the left, or pick a saved payee to auto-fill it.
  • Click + Add row to type straight into the table.
  • Paste straight from Excel — select your cells, copy, then press Cmd/Ctrl + V on the Build batch screen. Rows load automatically.
  • Click Import… to load a CSV (e.g. an Excel export). Click Template first to get a correctly-formatted file to fill in.

3. Fix any errors inline

Every cell in the table is editable. Invalid cells are outlined in red with the reason shown underneath; cells that will be auto-adjusted are flagged in amber. The Export button stays disabled until every row is valid.

4. Export

Click Export, choose where to save, and the file opens automatically so you can eyeball it before uploading to Santander Connect.

Validation rules

FieldRule
Beneficiary nameRequired. Bacs: uppercased, max 35 (first 18 reach Bacs). Allowed: A–Z 0–9 . - / & space.
Sort codeExactly 6 digits (dashes/spaces are ignored).
Account numberExactly 8 digits.
AmountGreater than 0, pounds & pence (e.g. 150.50).
ReferenceMax 18 chars. Mandatory for MULTIBACS; optional otherwise.
RTI referenceBacs only, optional. Starts with “/”, e.g. /123.

Your data is encrypted

Payees, saved batches and settings are stored only on this computer, encrypted with your operating-system keychain. Nothing is ever sent over the internet.

Back up & restore

The encrypted store is locked to this computer, so it can’t be copied to a new machine as-is. Save a backup file to move your data, or to keep a safe copy. The backup is encrypted with a password you choose (AES-256), so the file is unreadable to anyone without it — but keep the password safe, as it can’t be recovered.

Important

Always do one small test upload in Santander Connect to confirm a format is accepted before relying on it for a full payment run.

FAQ

My bank rejected the file — what now?

First, always do one small test upload before a full run. If a column name, order or value is off, it's a quick fix — the whole format lives in one place and can be adjusted. Double-check you picked the right output format (Bacs import vs Mixed) for your channel.

Why is the Export button greyed out?

There's at least one invalid row. Look for cells outlined in red and fix them — Export enables once every row is valid.

What's the amber "modulus check" warning on an account?

PayBatch runs the official UK bank modulus check (VocaLink / Pay.UK) on the sort code + account number. An amber warning means that combination can't be a real account — almost always a typo. It's a warning, not a block: double-check the digits, but you can still export if you're sure they're right.

Can I paste from Excel or Google Sheets?

Yes. Select the cells, copy, then press Cmd/Ctrl + V on the Build batch screen. Columns are detected automatically. A header row is optional.

What's the difference between BACS and MULTIBACS?

Single (BACS) is for one or more independent payments. Multiple (MULTIBACS) is for many payments that share one debit account and payment date — and every payment then needs a reference.

Is my data sent anywhere?

No. Everything stays on this computer and is encrypted with your operating-system keychain. PayBatch makes no network requests except an optional check for app updates. The storage location is shown in the bottom bar.

Can I pay in euros (SEPA)?

Yes. Choose the ISO 20022 bank, then the SEPA EUR format. The grid switches to IBAN / BIC columns; IBANs are checksum-checked, BIC is optional. The file is SEPA pain.001.001.03. UK domestic (GBP) and SEPA (EUR) are separate formats — do a test upload to confirm your bank's SEPA profile.

Which banks are supported?

Santander is available now. Lloyds, Barclays, HSBC and NatWest are on the roadmap and appear as “coming soon” on the home screen.

How do I get a new version?

When an update is available you'll see a banner at the top with a Download link. You can also click Check for updates below.

Something went wrong — how do I report it?

If an error appears it includes a short reference code (e.g. ERR-AB12C). Note that code; you can view recent issues and copy diagnostics from Report a problem below.

Report a problem

If something goes wrong, PayBatch records a local error with a short reference code you can quote when asking for help. Nothing is sent anywhere — the log stays on this computer.
